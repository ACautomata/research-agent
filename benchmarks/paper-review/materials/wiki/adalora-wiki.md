# AdaLoRA: Adaptive Budget Allocation for Parameter-Efficient Fine-Tuning

## 0. 元信息
- 标题：AdaLoRA: Adaptive Budget Allocation for Parameter-Efficient Fine-Tuning

- 年份：2023

- 研究方向关键词：Parameter-Efficient Fine-Tuning, Low-Rank Adaptation, Adaptive Budget Allocation, Singular Value Decomposition

## 1. 研究背景

大型预训练语言模型（PLMs）在下游任务上的微调已成为 NLP 的重要范式。然而，常见的全量微调需要为每个任务维护一份完整模型的独立副本，当存在大量下游任务时，内存开销变得极其高昂。LoRA 通过两个小矩阵 BA 的乘积参数化增量更新，参数量通常少于全量微调的 0.5%，训练开销可降低 70%，同时性能可与全量微调媲美。然而，LoRA 为所有增量矩阵分配相同的秩 r，忽略了不同模块和层间权重矩阵的重要性差异——微调 FFN 比微调 Self-Attention 更重要，顶层 Transformer 层比底层更重要。AdaLoRA 提出自适应地在不同权重矩阵之间分配参数预算，使重要的增量矩阵获得更高的秩。

## 2. 任务定义

给定一个预训练模型，其权重矩阵为 W^(0)，目标任务是在保留预训练知识的同时，以参数高效的方式适配下游任务。对于每个预训练权重矩阵 W^(0)，学习一个增量更新 Δ，使得 W = W^(0) + Δ。目标是在给定的参数预算（即总可训练参数量）下，最大化下游任务的性能。AdaLoRA 进一步要求：自适应地在不同权重矩阵之间分配参数预算，使得重要的增量矩阵获得更高的秩，而不重要的增量矩阵被剪枝至更低的秩。

## 3. 论文要解决的核心问题

**核心问题**：如何在 LoRA 类低秩微调方法中，根据权重矩阵的重要性自适应地分配参数预算，以取代均匀分配策略？论文通过实验明确了两个关键观察：微调 FFN 比微调 Self-Attention 模块带来的性能提升更大；顶层 Transformer 层的权重矩阵比底层更重要。均匀分配预算（如 LoRA）会导致次优性能——向关键权重矩阵分配更多参数可提升性能，而向不重要的权重矩阵增加参数则收益甚微。

## 4. 方法总览

AdaLoRA 将增量更新参数化为 SVD 形式 Δ = PΛQ，然后根据新颖的重要性指标动态剪枝奇异值，从而自适应地在各权重矩阵间分配参数预算。两个核心组件：(1) SVD 参数化——将增量矩阵参数化为 PΛQ 的形式，模拟奇异值分解但避免昂贵的精确 SVD 计算；(2) 重要性感知的秩分配——将 Δ = PΛQ 划分为三联组（triplets），每个三联组包含一个奇异值和对应的奇异向量，基于敏感性和不确定性量化计算重要性分数，然后根据全局预算调度器剪枝低重要性三联组的奇异值。

## 5. 方法关键模块

### 5.1 SVD 参数化
- P ∈ R^(d1×r)：左奇异向量
- Λ ∈ R^(r×r)：对角矩阵（含奇异值 λ_i），实际仅保存为向量
- Q ∈ R^(r×d2)：右奇异向量
- 初始时 Λ 初始化为零，P 和 Q 采用随机高斯初始化，使得 Δ = 0 在训练开始时成立
- 正交正则化项 R(P, Q) = ||P^T P - I||_F^2 + ||Q Q^T - I||_F^2 保持 P 和 Q 的正交性

### 5.2 重要性感知的秩分配
- 三联组 G_{k,i} = {P_{*,i}, λ_i, Q_{i,*}} 的重要性分数 S_{k,i} 综合考量敏感性与不确定性
- 敏感性 I(w) = |w * ∇_w L|（近似移除该参数后的损失变化）
- 平滑后 I^(t)(w) = β1 * I^(t-1)(w) + (1-β1) * I^(t)(w)
- 不确定性 U^(t)(w) = β2 * U^(t-1)(w) + (1-β2) * |I^(t)(w) - I^(t-1)(w)|
- 最终重要性 s^(t)(w) = I^(t)(w) * U^(t)(w)

### 5.3 全局预算调度器
- 三段式调度：预热阶段（保持初始预算 1.5×）→ 立方调度下降阶段 → 固定阶段
- 预算 b(t) 定义为所有增量矩阵的总秩（总奇异值数量）

## 6. 关键公式与机制说明

**SVD 参数化更新**：W = W^(0) + PΛQ

**训练损失**：L(P, E, Q) = C(P, E, Q) + γ * Σ_k R(P_k, Q_k)，其中 C 是原始训练损失，γ > 0 是正则化系数

**三联组重要性**：S_{k,i} = s(λ_{k,i}) + avg_j s(P_{k,ji}) + avg_j s(Q_{k,ij})

**立方调度**：b(t) = b(T) + (b(0)-b(T)) * (1 - (t-ti-tf)/(T-ti-tf))^3

**奇异值剪枝**：将不属于全局 top-b(t) 的奇异值置零

## 7. 训练与推理流程

**训练算法**：初始化 P（随机高斯）、Λ（零）、Q（随机高斯），为所有权重矩阵设置初始秩。每步：采样 mini-batch 计算梯度 → 计算每个参数的敏感性 → 更新平滑敏感性和不确定性 → 计算三联组重要性分数 → 更新 P 和 Q → 根据预算 b(t) 和重要性分数剪枝 Λ。奇异值剪枝每 ΔT 步执行一次（默认 ΔT=100），以便被剪枝的三联组有机会在间隔期内被重新激活。LoRA 的 scaling 因子 α/r 同样应用于 AdaLoRA。

**推理流程**：与 LoRA 相同——将训练好的增量矩阵合并回预训练权重，不引入额外推理开销。

## 8. 实验设置

- **模型**：DeBERTaV3-base（183M 参数，NLU + QA）、BART-large（NLG）
- **数据集**：GLUE（8 子任务）、SQuAD v1.1/v2.0、XSum、CNN/DailyMail
- **Baselines**：Full FT、BitFit、HAdapter、PAdapter、LoRA（r=2/4/8）
- **预算水平**：GLUE 0.3M/0.6M/1.2M；SQuAD 0.08%-0.65%；NLG 0.13%-2.20%
- **训练配置**：PyTorch + Huggingface Transformers，NVIDIA V100 GPUs，AdamW，学习率各任务独立调优，β1=β2=0.85，剪枝间隔 ΔT 多数任务 100
- **GLUE 结果报告 5 次随机种子均值，显著性检验 p < 0.05

## 9. 主要实验结果

**GLUE（Table 1）**：AdaLoRA 在所有 8 个子任务上、所有预算水平下一致超越所有基线。0.3M 低预算下平均 88.86 vs LoRA(r=2) 88.15，RTE 上高 1.8%。0.3M 预算下 CoLA 70.04 Mcc 超越所有基线在 0.6M 和 1.2M 预算下的结果。

**SQuAD（Table 2）**：AdaLoRA 在所有预算水平上均超越 LoRA。0.08% 预算下 SQuADv2.0 F1=88.7，接近 0.65% 预算下的 88.9，且超过全量微调的 88.4。

**NLG（Table 3）**：XSum 上增益显著（1.10% 预算 R-2: 21.13 vs 19.89），CNN/DailyMail 上增益较小，2.20% 预算下 LoRA 在 ROUGE-1 上反超 0.03。

**消融实验**：SVD 参数化优于 LoRA 结构化剪枝；完整重要性指标（敏感性+不确定性）> 仅敏感性 > 仅奇异值幅度；两个组件均有贡献。

**训练代价**：比 LoRA 增加约 11-16% 训练时间，GPU 显存差距小于 0.1 GB。未在 decoder-only 架构（GPT/LLaMA）上验证。

## 10. 论文贡献总结

1. 提出 AdaLoRA，首个在低秩微调中实现自适应参数预算分配的方法
2. SVD 参数化框架：通过剪枝奇异值来控制秩，避免昂贵的精确 SVD 计算
3. 新颖的重要性指标：结合敏感性平滑和不确定性量化
4. 全局预算调度器：引入立方调度的三段式预算调度策略
5. 广泛的实验验证：3 类任务 × 2 个模型 × 12 个数据集，代码和数据开源

## 11. 方法特点总结

**优势**：自适应预算分配使低预算下增益尤其显著；SVD 参数化可恢复（仅置零奇异值而保留奇异向量）；正交正则化稳定训练；训练代价可控（+11-16% 时间，显存几乎不变）；无需额外推理开销。
**局限**：仅验证了 DeBERTaV3-base 和 BART-large 两种模型，未在 decoder-only 架构（GPT/LLaMA）上测试；CNN/DailyMail 增益较小；重要性指标消融仅覆盖 3 个 NLU 子任务；未对比 Diff Pruning、Prefix Tuning 等其他 PEFT 方法。

## 12. 术语与概念表

| 术语 | 定义 |
|------|------|
| PEFT | 参数高效微调，仅更新少量额外参数来适配下游任务 |
| LoRA | 低秩适应，通过两个小矩阵 BA 的乘积参数化增量更新 |
| SVD 参数化 | 将增量矩阵参数化为 PΛQ 形式以模拟奇异值分解 |
| 三联组（Triplet） | 单个奇异值及对应左右奇异向量构成的组 |
| 重要性分数 | 衡量三联组对模型性能贡献的量化指标 |
| 敏感性 | I(w) = \|w · ∇_w L\|，近似移除参数后的损失变化 |
| 不确定性量化 | 通过局部变动衡量敏感性估计的可信度 |
| 立方调度 | 预算按照立方函数从初始值下降到目标值 |
| 正交正则化 | 用于保持奇异向量矩阵正交性的正则化项 |

## 14. 适合后续研究时重点关注的内容

1. **跨架构泛化性**（最优先）：仅在 DeBERTaV3-base 和 BART-large 上验证，未在 GPT/LLaMA/T5 等更广泛架构上测试
2. **重要性指标的跨任务消融**：指标消融仅覆盖 3 个 NLU 子任务，未在 QA 和 NLG 上验证
3. **CNN/DailyMail 上增益极小的根因分析**：与 XSum 上的显著增益形成对比
4. **超参数敏感度**：β1、β2 固定为 0.85 未做扫描；ΔT 未做系统对比；立方调度 vs 线性/余弦调度未做对比
5. **SVD 参数化 vs 自适应分配的独立贡献分离**：SVD-LoRA 已优于 LoRA，自适应分配的净增益需要 2×2 消融矩阵确认
6. **等计算预算公平对比**：AdaLoRA 训练 2-3× LoRA 时间，固定总 GPU-hours 预算下对比尚未完成
7. **与其他 PEFT 方法的全景对比**：未对比 Diff Pruning、Prefix Tuning 等

## 15. 一句话总结

AdaLoRA 提出了一种基于 SVD 参数化和重要性感知秩分配的自适应预算分配方法，在参数高效微调中取代了 LoRA 的均匀分配策略，在 12 个数据集上一致超越基线方法，且仅增加 11-16% 的训练时间。
