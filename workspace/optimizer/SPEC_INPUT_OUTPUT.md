# Optimizer Agent 全流程输入输出说明

## 概述

Optimizer agent 负责分析模型架构瓶颈，生成架构优化方案。独立 agent，不依赖上游流水线。

## 1. 输入

### 必选

| 项目 | 格式 | 来源 |
|------|------|------|
| 代码仓库路径或结构说明 | 文本 | 用户 |
| 模型定义文件 | Python / YAML / JSON | 用户代码 |

### 可选

| 项目 | 格式 | 来源 |
|------|------|------|
| 当前性能数据 | 数值 | 用户/自测 |
| 优化目标 | 文本 | 用户 |
| 约束条件 | 文本 | 用户 |

## 2. 处理流程

```
用户请求 → Step 1 诊断(A)→ Step 2 选方向(B)→ Step 3 确认(C)→ 输出方案
```

阶段 A（诊断）：读模型代码 → 识别瓶颈 → 输出诊断报告
阶段 B（方案）：策略匹配 → 生成方案对比 → 用户选择
阶段 C（产出）：细化方案 → 评估兼容性 → 生成最终文档

## 3. 输出

| 项目 | 说明 |
|------|------|
| 文件名 | `{项目名称}-architecture-optimization-plan.md` |
| 格式 | Markdown，UTF-8 |
| 默认路径 | `outputs/{项目名称}/{项目名称}-architecture-optimization-plan.md` |

### 内容结构：7 章节

## 4. 上下游关系

```
用户代码 → optimizer → spec → claude-code → 执行架构修改
                  ↓
            judge 验证效果
```

## 5. 边界

- 不写代码、不调数值参数、不做论文分析、不运行实验
- 架构优化和参数调优分离（各自输出独立方案）
