# Checklist 交互范式

用于需要用户逐项确认的 agent（tuning / optimizer）。

## 通用交互流程

```
Step 1: 用户提供输入材料
    │
    ├─ 返回《检查清单》→ 用户勾选/确认
    │
Step 2: 用户做出选择
    │
    ├─ 返回《选项对比表》→ 用户选择
    │
Step 3: 用户确认
    │
    ├─ 返回《确认清单》→ 用户确认
    │
Step 4: 生成最终方案
    │
    ├─ 输出方案文档
```

## 通用检查项模板

| 步骤 | 检查项 | 说明 |
|------|--------|------|
| Step 1 | [ ] 输入已理解 | 用户提供的材料已全部读取 |
| Step 1 | [ ] 约束已明确 | 预算、时间、资源限制已记录 |
| Step 2 | [ ] 方案已了解 | 每个方案的收益和风险已评估 |
| Step 2 | [ ] 选择已做出 | 用户已选择要实施的方案 |
| Step 3 | [ ] 兼容性已验证 | 方案与现有系统兼容 |
| Step 3 | [ ] 回滚方案已准备 | 如果效果不好可以回退 |

## 适用条件

- **需要用户决策**：方案选择和实施路径需要用户确认
- **风险较高**：改动的潜在影响较大，需要用户知情
- **成本较高**：方案执行需要较多资源，需要用户同意

## 不适用条件

- 输入输出固定，中间不需要用户干预 → 使用 BlackBox 范式
- 用户明确要求"直接做，不要问"
