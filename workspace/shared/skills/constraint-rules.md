# 通用约束规则

所有产出文档类的 agent（spec / tuning / optimizer）应遵守以下约束规则。

## 通用约束

| # | 约束 | 说明 | spec | tuning | optimizer |
|---|------|------|------|--------|-----------|
| 1 | 不写代码 | 只生成方案文档，不修改代码文件 | ✅ | ✅ | ✅ |
| 2 | 不运行实验 | 不执行训练/评估命令 | ✅ | ✅ | ✅ |
| 3 | 不臆造信息 | 缺失数据使用占位符，不编造 | ✅ | ✅ | ✅ |
| 4 | 标注置信度 | 建议标明：强推荐/推荐/仅供参考 | ⚠️ | ✅ | ✅ |
| 5 | 标注来源 | 参数范围/数据来源诚实标注 | ✅ | ✅ | ✅ |
| 6 | 最小侵入 | 生成方案时强调复用现有代码 | ✅ | ⚠️ | ⚠️ |
| 7 | 不扩大范围 | 只翻译上游给定的任务，不擅自添加 | ✅ | ✅ | ✅ |
| 8 | 不回写 wiki | 不创建或修改 wiki 页面 | ✅ | ✅ | ✅ |

## 通用约束规则详细说明

### 1. 不写代码
生成的方案文档不应包含可直接执行的代码，而是面向 claude-code 或用户的编码任务说明。

### 2. 不运行实验
agent 通过工具只做分析（read / exec / web_search），不主动运行训练或评估。

### 3. 不臆造信息
- 缺失路径 → `此处应填写...`
- 缺失范围 → `待确认`
- 缺失数据 → `需要先跑一次基线`
- 不确定的推断 → `理论推断，需实验验证`

### 4. 标注置信度
每个建议标明：
- **强推荐**：有充分理论或实验依据
- **推荐**：有较好依据但需验证
- **仅供参考**：理论推断，缺少直接证据

### 5. 标注来源
- 代码中读取的默认值：标注"来自代码"
- 经验范围：标注"经验范围"
- 论文参考：标注"参考文献 [X]"
- 用户提供：标注"用户提供"
