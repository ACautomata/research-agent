# 缺失信息处理策略

当输入材料不完整时，按以下策略处理。

## 信息分类

| 信息类型 | 处理方式 | 示例 |
|---------|---------|------|
| **路径** | 使用占位符 | `此处应填写训练入口文件路径` |
| **数值** | 标注"待确认"或使用经验值 | `[学习率]: 经验范围 1e-5 ~ 1e-1（待确认）` |
| **来源** | 标注来源类型 | `来自代码默认值` / `经验范围` / `参考文献` |
| **推断** | 标注置信度 | `理论推断，需实验验证` |
| **缺失材料** | 在输出中明确标注 | `⚠️ 缺失信息：问题分析文档未提供` |

## 占位符列表

| 场景 | 占位符 |
|------|--------|
| 代码仓库路径 | `此处应填写代码仓库路径` |
| 训练入口 | `此处应填写训练入口文件路径` |
| 配置文件 | `此处应填写配置文件路径` |
| 数据来源 | `此处应填写数据加载文件或目录` |
| 模型定义 | `此处应填写模型文件或目录` |
| 参数范围 | `待确认：请用户补充搜索范围` |
| 基线数据 | `需要先跑一次基线获取当前性能` |

## 原则

1. **禁止臆造** —— 路径、数字、范围必须有来源
2. **明确标注** —— 缺失信息在输出文档中标出，不让下游猜测
3. **保持模板完整** —— 即使信息缺失，输出文档的章节结构不应缺失
4. **给出最小可行方案** —— 在信息不足时仍能生成可用的部分方案
