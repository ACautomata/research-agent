# 输出文档章节结构规范

所有产出文档类的 agent 应遵循统一的章节结构框架。

## 通用章节结构

| 章节类型 | 是否必选 | 内容 | 适用 agent |
|---------|---------|------|-----------|
| 任务背景/优化目标 | 必选 | 描述任务目标、基线、约束 | 全部 |
| 输入材料清单 | 必选 | 列出可用输入及缺失信息 | spec |
| 详细方案 | 必选 | 核心方案内容 | 全部 |
| 收益/成本评估 | 推荐 | 方案的利益和代价 | tuning / optimizer |
| 实施路径 | 推荐 | 分阶段执行计划 | tuning / optimizer |
| 回滚方案 | 推荐 | 如何回退改动 | optimizer |
| 验证方法 | 推荐 | 如何验证方案效果 | 全部 |
| 交付物说明 | 必选 | 输出什么、保存到哪里 | spec |
| 完成标准 | 必选 | 判断任务完成的条件 | spec / tuning |
| 自检清单 | 内部使用 | 输出前逐项确认 | 全部 |

## 命名规范

| 项目 | 规范 |
|------|------|
| 文件名 | `{项目名称}-{类型}.md` |
| 类型后缀 | spec: `codex-prompt` / tuning: `tuning-plan` / optimizer: `architecture-optimization-plan` |
| 默认路径 | `outputs/{项目名称}/{文件名}` |
| 编码 | UTF-8 |
