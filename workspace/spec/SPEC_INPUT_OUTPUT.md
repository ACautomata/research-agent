# Spec Agent 全流程输入输出说明

## 概述

Spec agent（pipeline S5）位于论文自动化分析流水线的第 5 阶段。它的职责是将前序阶段产生的问题分析与验证实验设计，翻译成 claude-code 可直接执行的工程任务提示词。

---

## 1. 输入

### 1.1 必选输入

| 项目 | 格式 | 来源 | 说明 |
|------|------|------|------|
| 问题分析文档 | `*-problem.md` (Markdown) | S3 critic agent | 论文的潜在问题、可疑点、待验证判断 |
| 验证实验设计文档 | `*-validation.md` (Markdown) | S4 design agent | 针对上述问题的验证实验方案 |

### 1.2 可选输入

| 项目 | 格式 | 来源 | 说明 |
|------|------|------|------|
| 论文基础 Wiki 文档 | Markdown | S1 ingest agent | 提供论文背景上下文 |
| 代码仓库路径或结构说明 | 文本路径 / 目录树 | 用户 | 目标代码仓库的位置和结构 |

### 1.3 输入要求

- 所有 Markdown 文档使用 UTF-8 编码
- 问题分析文档应包含明确的"问题陈述 → 判断依据 → 待验证状态"结构
- 验证实验设计文档应包含"实验名称 → 目标 → 方法 → 评估指标"结构
- 代码仓库路径应为绝对路径或相对路径

---

## 2. 处理流程

### 阶段 A：输入接收与验证

```
输入材料到达
    │
    ▼
检查必选文件是否存在
    │
    ├─ ✅ 存在 → 进入阶段 B
    └─ ❌ 缺失 → 标注缺失信息，使用占位符
```

### 阶段 B：材料理解与映射

```
读取问题分析文档
读取验证实验设计文档
读取论文 Wiki（如提供）
读取代码结构说明（如提供）
    │
    ▼
映射关系：问题 → 实验 → 代码改动点
    │
    ▼
标记实验优先级（按最小改动优先）
```

### 阶段 C：提示词生成

```
按 SKILL.md 的输出模板填充内容
    │
    ▼
可用输入材料部分 → 章节 2
论文方法与验证目标 → 章节 3
优先实现的实验 → 章节 4
代码位置 → 章节 6
    │
    ▼
输出前自检（SKILL.md 章节 7 的 checklist）
```

### 阶段 D：输出交付

```
┌─────────────────────────────────────┐
│ 生成的文件：{论文简称}-codex-prompt.md │
│ 标题：# 发给 claude-code 的完整任务提示词 │
│ 内容：10 个结构化章节                   │
└─────────────────────────────────────┘
```

---

## 3. 输出

### 3.1 标准输出

| 项目 | 说明 |
|------|------|
| 文件名 | `{论文简称}-codex-prompt.md` |
| 格式 | Markdown，UTF-8 编码 |
| 默认路径 | `outputs/{论文简称}/{论文简称}-codex-prompt.md` |
| 标题要求 | `# 发给 claude-code 的完整任务提示词` |

### 3.2 输出内容结构

| 章节 | 内容 | 说明 |
|------|------|------|
| 1. 任务背景 | 验证性实验任务说明 | 明确这不是重写方法，不是重构 |
| 2. 可用输入材料 | 路径列表 | 代码仓库、Wiki、分析文档、设计文档 |
| 3. 论文方法与验证目标 | 模块 + 问题列表 | 只保留编码必要的信息 |
| 4. 优先实现的验证实验 | 3 个实验的详细描述 | 每个含：目标、类型、选项、实现要求 |
| 5. 总体实现要求 | 7 条编码规范 | 最小侵入、复用、配置化等 |
| 6. 建议优先查看的代码位置 | 8 个关键文件类别 | 训练入口、配置、数据加载等 |
| 7. 交付内容 | 5 类交付物 | 代码改动、入口、结果、说明、限制 |
| 8. 完成标准 | 9 条验收条件 | 用于判断任务是否完成 |
| 9. 汇报格式 | 5 个汇报板块 | 改动概览、文件列表、运行方式等 |
| 10. 一句话目标总结 | 任务一句话总结 | 收尾 |

### 3.3 缺失信息处理

如果输入材料未提供某路径或信息，使用以下占位符替代：
- `此处应填写代码仓库路径`
- `此处应填写论文基础 Wiki 路径`
- `此处应填写问题分析文档路径`
- `此处应填写验证实验设计文档路径`
- `请根据仓库结构自行定位`

---

## 4. 与其他 agent 的关系

```
上游                        本 agent                  下游
─────────                   ────────                  ──────
S3 critic ──问题分析──┐
                      ├──→ spec ──任务提示词──→ claude-code 执行代码实现
S4 design ──实验设计──┘
                                              claude-code ──代码改动──→ 原始代码仓库
```

- **不依赖**下游 claude-code 的执行结果
- **不修改**上游 S3、S4 的文件
- **不反向依赖**任何其他 agent

---

## 5. 边界与限制

| 边界 | 说明 |
|------|------|
| 不写代码 | 只生成让 claude-code 写代码的提示词 |
| 不运行实验 | 不执行任何训练或测试命令 |
| 不做论文分析 | 不分析论文贡献、不做文献综述 |
| 不扩大范围 | 只翻译上游给定的实验，不擅自添加 |
| 不编排他人 | 不调用 sessions_spawn，不委派其他 agent |
| 不修改 wiki | 不创建或编辑 wiki 页面 |
