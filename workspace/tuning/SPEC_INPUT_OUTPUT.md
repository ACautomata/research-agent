# Tuning Agent 全流程输入输出说明

## 概述

Tuning agent 负责分析算法代码的调参需求，制定搜索策略，生成调参优化方案。它是一个独立的 agent，不依赖上游流水线结果，可以直接接收用户请求。

---

## 1. 输入

### 1.1 必选输入

| 项目 | 格式 | 来源 | 说明 |
|------|------|------|------|
| 代码仓库路径或结构说明 | 文本路径 | 用户 | 要调参的目标代码 |
| 训练脚本入口和参数列表 | 文本 / CLI | 用户 | 从代码中提取的参数定义 |

### 1.2 可选输入

| 项目 | 格式 | 来源 | 说明 |
|------|------|------|------|
| 模型/算法配置文件 | YAML / JSON / 文本 | 用户 | 现有配置，含参数默认值 |
| 性能评估指标定义 | 文本 | 用户 | 如何评估模型性能 |
| 基线性能数据 | 数值 | 用户/自测 | 当前参数的性能基准 |
| 调参约束条件 | 文本 | 用户 | 时间预算、计算资源、精度要求 |

### 1.3 输入要求

- 代码仓库路径应为绝对路径或可访问的相对路径
- 参数列表应包含参数名、类型、当前值（如有）
- 评估指标应有明确的定义和计算方式
- 约束条件应可量化

---

## 2. 处理流程

### 交互流程（Checklist 模式）

```
用户请求调参
    │
    ▼
Step 1：参数识别
├── 读取代码仓库/配置文件，提取参数列表
├── 识别参数类型（连续/离散/整数/类别）
├── 确定参数当前值和源代码位置
└── 输出《调参参数清单》给用户确认
    │
    ▼
Step 2：搜索空间定义
├── 为每个参数推荐搜索范围
├── 标注重要性等级（高/中/低）
├── 将经验范围和代码默认值结合起来
└── 用户确认/调整搜索范围
    │
    ▼
Step 3：策略推荐
├── 根据 D（参数维度）和 N（预算）推荐策略
├── 至少给出 2 种方案可选
├── 每种方案估算计算预算
└── 用户选择策略
    │
    ▼
Step 4：评估方案确认
├── 定义评估指标
├── 确定验证方式（验证集/CV/重复实验）
├── 确认重复次数和随机种子
└── 用户确认
    │
    ▼
Step 5：方案生成
└── 输出完整调参优化方案文档
```

### 内部处理阶段

```
阶段 A：代码分析
├── exec 遍历代码仓库结构
├── read 读取配置文件和训练脚本
└── 提取参数定义和默认值

阶段 B：参数分类
├── 按类型分组（连续/离散/整数/类别）
├── 标注已知的经验范围和重要性
└── 生成参数清单

阶段 C：策略匹配
├── 计算 D（参数维度数）
├── 评估 N（可用预算）
├── 按规则匹配最优策略
└── 计算预算

阶段 D：方案组装
└── 按 SKILL.md 模板生成输出文档
```

## 3. 输出

### 3.1 标准输出

| 项目 | 说明 |
|------|------|
| 文件名 | `{项目名称}-tuning-plan.md` |
| 格式 | Markdown，UTF-8 编码 |
| 默认路径 | `outputs/{项目名称}/{项目名称}-tuning-plan.md` |

### 3.2 输出内容结构

| 章节 | 内容 | 说明 |
|------|------|------|
| 1. 优化目标 | 调参目标和基线 | 量化定义 |
| 2. 待调参数清单 | 完整参数表 | 含类型、范围、重要性 |
| 3. 搜索策略推荐 | 2-3 种策略对比 | 含预算估算 |
| 4. 评估方案 | 指标+验证方式 | 确保可靠性 |
| 5. 执行计划 | 三阶段执行 | 粗搜→细搜→验证 |
| 6. 结果记录模板 | JSON 格式 | 可复现 |
| 7. 可视化建议 | 图表类型 | 结果分析 |

### 3.3 缺失信息处理

- 参数范围参考代码默认值或同类论文
- 无法确定时标注"参考值，待确认"
- 基线数据没有时不猜测，标注"需要先跑一次基线"

## 4. 与其他 agent 的关系

```
上游                             本 agent                    下游
─────────                       ────────                    ──────
用户/代码仓库 ──配置文件──→      tuning ──调参方案──→ spec → claude-code → 修改代码
用户 ──调参目标/约束──→                                          │
                                                                 ▼
                                                          执行调参实验
                                                                 │
                                                                 ▼
                                                           judge 验证结果
```

- **上游依赖**：代码仓库和配置文件（用户提供）
- **下游依赖**：spec + claude-code 执行调参代码修改
- **可选依赖**：web_search 搜索调参参考资料

---

## 5. 边界与限制

| 边界 | 说明 |
|------|------|
| 不写代码 | 只生成调参方案，不修改代码文件 |
| 不运行实验 | 不执行训练或评估命令 |
| 不做论文分析 | 不分析论文贡献 |
| 不验证结果 | 调参结果由 judge agent 验证 |
| 不臆造数据 | 参数范围标注来源 |
| 推荐 ≥ 2 种策略 | 不固守一种方法 |
